## 一、概述

### 1.1 系统定位

QooERP是一款轻量级微服务架构的开源ERP系统，支持：

#### 适用对象
- **大型企业**：集团公司、跨国企业（支持百万级用户）
- **政府机构**：中央-省-市-县-乡五级架构（支持千万级用户）
- **科技企业**：互联网公司、高新技术企业（支持百万级并发）
- **中小企业**：中小企业、小微企业（兼容部署）

#### 核心能力
- **弹性扩展**：支持 100-1000000+ 用户规模，从小微企业到百万级用户弹性扩展
- **轻量级架构**：采用微服务架构，按需启用模块，避免过度设计
- **分布式能力**：支持分布式缓存、分布式事务、分库分表
- **多租户架构**：行隔离多租户，支持单实例多组织

#### 架构特性
- **数据库**：PostgreSQL + 分库分表 + 读写分离
- **领域驱动**：DDD领域驱动设计
- **微服务**：轻量级微服务架构，按业务域拆分
- **多租户**：行隔离多租户架构

### 8.2 架构特性

| 特性 | 方案 | 说明 |
|------|------|------|
| 微服务 | 轻量级微服务 | 按业务域拆分，按需启用模块 |
| 多租户 | 行隔离 | TenantID字段，支持单实例多组织 |
| 分布式事务 | Seata分布式事务 | 支持AT/TCC/SAGA模式 |
| 缓存 | L1(Caffeine) + L2(Redis) | 二级缓存架构，提升命中率和性能 |
| 消息队列 | RocketMQ | 异步解耦、流量削峰、高吞吐 |
| 分库分表 | ShardingSphere | 水平分片、垂直分片、读写分离 |
| 分布式锁 | Redisson | 分布式场景并发控制 |
| 服务治理 | Sentinel | 限流、熔断、降级、系统保护 |
| 负载均衡 | LoadBalancer | 客户端负载均衡，支持多实例 |
| 服务调用 | OpenFeign | 声明式HTTP客户端，简化微服务调用 |
| 搜索 | PostgreSQL全文检索 | tsvector字段 + pgvector向量搜索 |
| 权限 | RBAC+ABAC | 复合权限模型，支持行级/列级/字段级权限 |
| 国际化 | i18n | 多语言支持，100+语言 |
| 监控 | Prometheus + Grafana | 全链路监控告警 |
| 链路追踪 | OpenTelemetry | 分布式链路追踪，定位性能瓶颈 |

### 8.3 部署方案

| 企业规模 | 部署方案 | 并发能力 | 架构特点 |
|----------|---------|---------|---------|
| 小微企业 | Docker一键部署 | 100-500用户 | 单体服务 + PostgreSQL + Redis |
| 中型企业 | 微服务集群部署 | 500-5000用户 | 微服务集群 + 二级缓存 + 分库分表 |
| 大型企业 | K8s集群部署 | 5000-50000用户 | 完整微服务 + 分布式缓存 + 分布式事务 + MQ |
| 集团/科技企业 | K8s多区域部署 | 50000-1000000+用户 | 多机房 + 多级缓存 + 异地多活 + 监控告警 |
| 政府机构 | 私有云部署 | 1000000+用户 | 等保三级 + 多级安全 + 完整审计 |
