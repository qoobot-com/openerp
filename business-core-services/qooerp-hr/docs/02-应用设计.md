## 一、应用架构设计

### 1.1 分层架构

```
┌─────────────────────────────────────────┐
│           Presentation Layer            │
│  (Controller, DTO, API Gateway Route)    │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           Business Layer                │
│  (Service, Domain Model, Business Logic) │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         Data Access Layer               │
│    (Repository, Mapper, DAO)            │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│         Infrastructure Layer            │
│  (Database, Cache, MQ, External API)    │
└─────────────────────────────────────────┘
```

### 1.2 模块划分

| 模块名称 | 职责说明 |
|---------|---------|
| hr-employee | 员工管理模块 |
| hr-attendance | 考勤管理模块 |
| hr-salary | 薪资管理模块 |
| hr-performance | 绩效管理模块 |
| hr-integration | 集成模块 |
