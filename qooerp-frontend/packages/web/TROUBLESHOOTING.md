# 主页空白问题排查指南

## 问题描述
访问主页时页面显示空白,没有任何内容。

## 可能的原因和解决方案

### 1. 依赖未安装
**症状**: 控制台显示模块未找到错误
**解决方案**:
```bash
cd d:/05workspaces/qooerp/qooerp-frontend
pnpm install
```

### 2. 后端服务未启动
**症状**: API 请求失败,网络错误
**解决方案**:
- 确保后端服务运行在 `http://localhost:8080`
- 或者修改 `vite.config.ts` 中的代理配置

### 3. Token 认证问题
**症状**: 路由守卫跳转到登录页
**解决方案**:
- 先访问 `/login` 页面进行登录
- 或者临时设置 `localStorage.setItem('token', 'test')`

### 4. 路由配置问题
**症状**: 路由匹配失败
**解决方案**:
- 检查浏览器控制台的路由跳转日志
- 确认路由守卫没有阻止渲染

### 5. 组件加载失败
**症状**: 组件渲染错误
**解决方案**:
- 检查 TDesign 组件是否正确注册
- 查看浏览器控制台的错误信息

## 调试步骤

1. 打开浏览器开发者工具 (F12)
2. 查看 Console 标签页的错误信息
3. 查看 Network 标签页的请求状态
4. 检查 Application 标签页的 LocalStorage

## 快速修复

如果页面仍然空白,尝试以下步骤:

1. 清除浏览器缓存和 LocalStorage
2. 停止开发服务器 (Ctrl+C)
3. 删除 node_modules 和 pnpm-lock.yaml
4. 重新安装依赖: `pnpm install`
5. 重新启动开发服务器: `pnpm run dev`
